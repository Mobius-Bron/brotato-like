二、设计模式

  1. ECS (Entity-Component-System)
     - Entity: 仅 ID 标识
     - Component: 纯数据结构，无逻辑
     - System: 遍历组件处理逻辑

  2. 工厂模式 (Factory Pattern)
     - EnemyFactory: 从 CSV 配置创建敌人实体
     - WeaponFactory: 从 CSV 配置创建武器实体
     - ProjectileFactory: 从对象池获取子弹实体

  3. 对象池 (Object Pool)
     - 敌人、子弹、粒子效果使用对象池
     - 避免频繁 GC，提升性能

  4. 组件模式 (Component Pattern)
     - 角色 = 组件组合
     - 不同敌人只是 CSV 配置不同

  5. 观察者模式 (Observer Pattern)
     - EventBus 事件系统解耦 UI 和逻辑
     - 事件: 受伤/死亡/波次变化/金币变化

  6. 单例模式 (Singleton)
     - GameManager, EventBus, ConfigLoader 为单例
